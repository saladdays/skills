# roundtable プラグイン開発ルール

## 概要

多分野の世界最高レベルの専門家を動的に選定し、エージェントチームで構造化された議論を行い、
多角的な評価・提言をまとめるプラグイン。コードレビューに限らず、Webサイト評価、事業戦略、
デザイン、組織設計等あらゆるドメインに対応。

## 重要ファイル

| ファイル | 役割 |
|---|---|
| `skills/start/SKILL.md` | スキル本体。4フェーズのパイプライン定義 |
| `skills/start/references/expert-archetypes.md` | 専門家アーキタイプ集（ドメイン→推奨専門家） |
| `skills/start/references/deliberation-rules.md` | 議論ルール（収束条件、重み付け、少数意見保護） |
| `skills/start/references/output-format.md` | 出力フォーマット |

## 設計思想

- **「ただ議論させるだけでは価値がない。構造設計が全て」**（ICLR 2025 MADサーベイ）
- 独立した意見生成を先にする（デルファイ法・NGT）
- 反対意見を構造的に保証する（Devil's Advocate）
- 反復は最大2ラウンド（3以上は逆効果: Degeneration of Thought）
- 少数意見を構造的に保護する（Groupthink回避）
- 自己申告の確信度は使わない（研究で過剰自信が確認済み）

## 開発方針

- SKILL.md がパイプラインの正
- expert-archetypes.md は原理ベース（キーワードリストではない）で設計
- 「同意しすぎ問題」への対策を常に意識する
- Claude Code / Cursor 両対応。SKILL.md ではツール固有の実装を指定しない

## テスト方法

以下のドメインでテスト：
- 技術系: Webサイト評価、コードアーキテクチャ
- 非技術系: 事業戦略、デザイン評価、組織設計
- 対立が生まれるケースで Phase 2 が正しく動作するか確認
- 少数意見が最終レポートに含まれるか確認
