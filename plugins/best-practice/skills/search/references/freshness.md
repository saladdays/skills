# 鮮度分類の詳細

## 二層ラベルモデル

一つのトピックに2つの鮮度を持たせる。

### Layer S（Stable Semantics / 安定意味層）
原則・理論・問いの立て方・トレードオフ。変化が遅い。

### Layer V（Volatile Surface / 流動表面層）
ツール名・テンプレURL・具体的手順・組織固有の用語。変化が速い。

**例:**
| トピック | Layer S | Layer V |
|---------|---------|---------|
| アジャイル運用 | マニフェストの4価値（foundational） | Jira設定・スクラムガイド版（operational） |
| PRDの書き方 | 構造化・仮説検証の原則（strategic） | Notionテンプレ・AIツール活用（tactical） |
| 認証のベストプラクティス | ゼロトラストの原則（strategic） | OAuth 2.1仕様・ライブラリ（tactical） |
| 子どもの学習支援 | 動機づけ・発達心理の原理（foundational） | 学習アプリ・教育制度（operational） |
| デザインシステム | 一貫性・再利用性の原則（strategic） | Figmaプラグイン・トレンド配色（tactical） |

## 鮮度クラスの詳細

**判断の原理**: 「この情報が間違っていたら、どれくらいの速さで問題が起きるか」で考える。
以下は各クラスの考え方と例。例は非網羅的であり、リストにないトピックでも
同じ原理（情報の変化速度と、古い情報に従った場合の影響度）で判断する。

### 1. Ephemeral（瞬間〜日次）
- **考え方**: 数日で無効になり、古い情報に従うと即座に問題が起きる
- **例**: 株価、障害速報、キャンペーン価格
- **検索**: リアルタイム検索、公式のみ

### 2. Tactical（週〜四半期）
- **考え方**: 頻繁に更新され、バージョンや時期によって正解が変わる
- **例**: API料金、CVE、ツールの具体的手順
- **検索**: 3ヶ月以内の結果を優先、公式の変更履歴を確認

### 3. Operational（6〜18ヶ月）
- **考え方**: ある程度安定しているが、メジャーアップデートで変わりうる
- **例**: フレームワークの推奨パターン、プロセス設計の具体的テンプレ
- **検索**: 版・バージョン・対象年度等との照合必須

### 4. Strategic / Canonical（2〜7年）
- **考え方**: パラダイムレベルで変わるもの。骨格は数年単位で入れ替わる
- **例**: 主流のマネジメント手法、デザインシステムの世代
- **検索**: 成熟度重視。事例は古くなるが考え方は残ることが多い

### 5. Foundational（10年以上）
- **考え方**: 原理・理論として長く効くが、研究の前沿は動いている
- **例**: アルゴリズム、発達心理の大枠、設計原則
- **検索**: 教科書的ソースも有効。ただし「常識」が覆る場合もある

### 6. Axiomatic（減衰なし）
- **考え方**: 定義上変わらない真実。このラベルは慎重に使う
- **例**: 数学的定理、物理法則
- **注意**: 「変わらないと思い込んでいるだけ」ではないか常に疑う

## 主張の型（Claim Type）

同じ鮮度クラスでも、主張の型によって危険度が変わる。

| 主張の型 | 説明 | 古いとどうなるか |
|----------|------|-----------------|
| **事実・数値** | 料金、法律条文、市場シェア | 即座に無効になりうる |
| **手順・How-to** | 設定方法、導入手順 | ツール・制度の変更で動かなくなる |
| **規範・合意** | アジャイルの価値、デザイン倫理 | 用語や議論は変わるが骨格は残る |
| **モデル・フレーム** | 3C、SWOT、学習理論 | 骨格は長く効く、事例は更新が必要 |
| **メタスキル** | 批判的思考、コミュニケーション | 最も長いが研究の前沿は動く |

## 主張の型が複合的な場合

一つのトピックに複数の主張の型が含まれる場合：
- **最も鮮度敏感な型を主要とする**（安全寄りの判断）
- 副次的な型も記録する
- 例: 「JWTの管理方法」→ 主要=手順（tactical）、副=規範（strategic）
  → Layer V は tactical 基準で検索する

出力例: `主張の型: 手順（主）、規範（副）`

## 鮮度判定のヒューリスティクス

1. クエリに年号・バージョン・「最新」がある → tactical 以下
2. クエリにツール名・サービス名がある → Layer V は tactical
3. クエリが「なぜ」「原則」「理論」→ Layer S は strategic 以上
4. クエリが「どうやって」「手順」「設定」→ Layer V は operational 以下
5. 判断に迷ったら operational をデフォルトとする（安全寄り）

## 検索時の鮮度適用

```
検索クエリに付与:
- ephemeral/tactical: "2026" or "2025" を追加、site: で公式指定
- operational: バージョン名を追加
- strategic: "best practices" "lessons learned" を追加
- foundational/axiomatic: 年指定しない
```

## メタデータテンプレート

各調査結果に付与する鮮度メタデータ：
```
freshness_layer_s: [クラス]
freshness_layer_v: [クラス]
claim_type: [事実/手順/規範/モデル/メタスキル]
last_verified_date: [確認日]
source_date: [ソースの公開/更新日]
target_version: [対象バージョン（あれば）]
```
