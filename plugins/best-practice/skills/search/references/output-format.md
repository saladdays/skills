# 出力フォーマットと不確実性の表現

## 最優先ルール（他のすべてに優先する）

**ソースに基づかない内容を生成してはならない。**
テンプレートの項目を埋めるために内容を作り上げるくらいなら、そのセクションを省略する。
省略した場合は「一次情報なしのため省略」と1行書く。

## 高影響トピックの免責注記（影響リスクが高い場合に必須）

影響リスクが高いトピックでは、レポート冒頭に免責注記を付ける。

```markdown
> **注意**: 本レポートは情報提供を目的としており、[医療/法的/金融/専門的]助言ではありません。
> 具体的な判断・行動の前に、必ず[医師/弁護士/税理士/専門家]にご相談ください。
```

領域に応じて文言を調整する。免責注記がないまま高影響トピックのレポートを出力しない。

## 基本原則

1. **内容の厚みに応じてフォーマットを調整する**: テンプレートは固定的に守るものではなく、
   実際に見つかった情報に合わせて項目を増減する。必要なら独自の項目を追加してもよい
2. **出典は必須**: すべての主張にインライン引用を付ける。引用できない主張は書かない
3. **不確実性は隠さない**: 断定と推測を文体で区別する
4. **アクション指向**: 結論だけでなく「次にやること」を含める

## フォーマットの考え方

テンプレートは参考構造であり、すべての項目を埋める義務はない。
各セクションは **必須** と **条件付き** に分かれる。
条件付きセクションは、該当するソースが見つかった場合のみ出力する。

### 必須セクション（常に出力）
- **エグゼクティブサマリ**: 結論・推奨（見つからなかった場合は「見つからなかった」と書く）
- **出典**: すべての主張の根拠URL

### 条件付きセクション（ソースがあるときのみ出力）
- **注意点・反パターン**: ソースに明示的に記載されている場合のみ
- **矛盾する情報**: 実際にソース間で矛盾がある場合のみ
- **専門視点別の評価**: 複数視点で検索した場合のみ
- **トレードオフ表**: 選択肢が複数ある場合のみ
- **プラクティスの成熟度**: 判断材料がある場合のみ
- **監査証跡**: Deep 深度の場合のみ（任意）

条件付きセクションを省略する場合、無言で省略してよい。
ただし、ユーザーが期待しそうな項目（例: 反パターン）を調べたが見つからなかった場合は
「調査したが一次情報なし」と1行添えると親切。

### Level 1: 簡潔回答（Quick 深度・単純な事実確認）

```markdown
## [トピック]

[結論を2-3行で]

**出典**: [URL] ([日付])
```

### Level 2: 標準レポート（Standard 深度）

必須セクション + 該当する条件付きセクションのみ。

```markdown
## [トピック] のベストプラクティス

### エグゼクティブサマリ
[結論・推奨を3-5行で。最も重要なことだけ]

### 推奨事項
- [推奨1] [1][2]
- [推奨2] [3]

### 出典
[1] [タイトル] ([ドメイン], [日付]) [URL]
[2] ...
```

### Level 3: 詳細レポート（Deep 深度）

```markdown
## [トピック] のベストプラクティス

### エグゼクティブサマリ
[結論・推奨・主要リスクを5行以内で]

### 推奨事項
#### [カテゴリ1]
- [推奨1] [1][2]
  - 根拠: [なぜこれが推奨されるか]
  - 適用条件: [どのような場合に有効か]

#### [カテゴリ2]
- [推奨2] [3][4]

### 注意点・反パターン
- [やってはいけないこと1] [5]
  - なぜ危険か: [理由]
  - 代わりにやるべきこと: [代替案]

### 専門視点別の評価（複数視点の場合）
#### [視点1: Security]
- [この視点からの推奨・懸念]

#### [視点2: UX]
- [この視点からの推奨・懸念]

#### トレードオフ
| 観点 | 選択肢A | 選択肢B |
|------|---------|---------|
| ... | ... | ... |

### 矛盾する情報（あれば）
- ソースA: [主張X] [1]
- ソースB: [主張Y] [3]
- 判断根拠: [...]

### 信頼度メタデータ
- 鮮度: Layer S=[クラス], Layer V=[クラス]
- ソース構成: Tier 1 x [N], Tier 2 x [N], Tier 3 x [N]
- 合意度: [高/中/低]（[根拠]）
- 検証状況: [実施した検証のサマリ]
- セキュリティ感度: [高/低]

### プラクティスの成熟度（該当する場合）
- ステータス: [🟢 Established / 🟡 Emerging / 🔴 Deprecated]
- 経緯: [いつ頃から推奨され始めたか、変遷があれば]

### 次のアクション
1. [具体的にやるべきこと]
2. [追加で確認すべきこと]

### 出典
[1] [タイトル] ([ドメイン], [日付]) [URL]
[2] ...
```

## 不確実性の表現パターン

### 断定レベル別の文体

| レベル | 文体 | 使う場面 |
|--------|------|---------|
| 確定 | 「〜である」「〜が推奨されている」 | Tier 1 で確認済み、複数ソースが一致 |
| 高確信 | 「〜とされている」「ドキュメント上は〜」 | Tier 1-2 で確認、他ソース未確認 |
| 中確信 | 「一般的には〜とされる」「多くの情報源が〜を推奨」 | Tier 2-3 で複数一致 |
| 低確信 | 「〜という見解がある」「コミュニティでは〜だが未検証」 | Tier 3-5、単一ソース |
| 未確認 | 「〜の可能性がある（未検証）」 | 裏取りができていない |
| 対立 | 「ソースAは〜、ソースBは〜。公式は〜を支持」 | 情報が矛盾 |

### 見つからなかった場合

```markdown
### 調査結果
[トピック]について、信頼性の高い情報源からの明確なベストプラクティスは
見つかりませんでした。

**検索した範囲:**
- [使用したクエリ1]
- [使用したクエリ2]
- [確認したソース]

**考えられる理由:**
- [このトピックはまだ確立されていない / ニッチすぎる / etc.]

**代替アプローチの提案:**
- [別のキーワードで検索する]
- [関連する隣接トピックを調べる]
- [特定のコミュニティに聞く]
```

## 監査証跡（調査プロセスの記録）

Deep 深度の場合、レポート末尾に調査プロセスを記録する（任意）：

```markdown
### 調査プロセス（参考）
- メタ判断: 鮮度=[クラス], 深度=[レベル], 言語=[戦略], 視点=[選定結果]
- 検索クエリ: [使用したクエリ一覧]
- 取得ソース: [N]件取得 → [M]件採用
- 不採用理由: [鮮度不足 x N, Tier不足 x N, 矛盾 x N]
- 検証実施: [実施した検証ステップ]
```
