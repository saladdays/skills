# Best Practice — ベストプラクティス調査

**「ベストプラクティス」を検索して、古い情報や個人の意見に振り回されたことはありませんか？**

技術選定、セキュリティ対策、価格設定——正しいやり方を知りたいのに、検索結果は玉石混交。3年前の記事が上位に来たり、ツールの宣伝がベストプラクティスとして紹介されていたり。

Search Best Practice はこの問題を解決します。

Web検索で最新のベストプラクティスを調査し、鮮度・信頼性・多角的視点を考慮した構造化レポートにまとめます。「原則（変わらないもの）」と「ツール・手法（すぐ変わるもの）」を分離して管理するので、来月には古くなる情報と、3年後も使える知見を区別できます。

---

## こんな方に

- **技術選定で迷っている方** — フレームワーク、ライブラリ、インフラ構成の「今の正解」を知りたい
- **新しいドメインに取り組む方** — 自分の専門外のベストプラクティスを効率よくキャッチアップしたい
- **チームの判断基準をつくりたい方** — 「なぜこのやり方がベストなのか」を根拠付きで説明したい
- **情報の鮮度が気になる方** — 古い記事に基づいて判断するリスクを避けたい

専門知識は必要ありません。知りたいことを伝えるだけで、調査から構造化までを自動で行います。

## 使い方

知りたいテーマを伝えるだけです。ドメインは自動で判定されます。

```
/best-practice:search Kubernetes シークレット管理のベストプラクティス
/best-practice:search SaaSの価格設定のベストプラクティス
/best-practice:search 子どもの読書習慣の身につけ方
```

### 調査の流れ

1. **自動判定** — テーマの鮮度要件、リスクレベル、必要な視点を自動で分析します
2. **Web調査** — 公式ドキュメント・学術論文・業界レポートを中心に、信頼性の高い情報源から調査します
3. **反証検索** — あえてアンチパターンや反対意見も探し、一面的な結論を防ぎます
4. **構造化レポート** — 原則とツールを分離した、すぐに使えるレポートが届きます

---

## レポートに何が書いてあるの？

| セクション | ひとことで言うと |
|---|---|
| **エグゼクティブサマリ** | 結論を簡潔に。いちばん大事なベストプラクティス |
| **原則（Layer S）** | 3年後も変わらない普遍的な原則。なぜそうすべきかの理由つき |
| **ツール・手法（Layer V）** | 今のベストな選択肢。鮮度と推奨度つき |
| **アンチパターン** | やってはいけないこと。よくある失敗とその理由 |
| **判断マトリクス** | 状況に応じた選択指針（「こういう場合はAを選ぶ」） |
| **情報源** | 参照した全ソースと信頼性評価 |

ポイントは、**原則とツールが分離されている**こと。「Kubernetes のシークレットは外部ストアに保存すべき」（原則 = 安定）と「Vault を使う」（ツール = 変わりうる）を区別できるので、ツールが変わっても判断基準は残ります。

## 対応ドメイン

技術・ビジネス・教育・医療・法務・デザイン・クリエイティブ等、あらゆるドメインに対応。原理ベースで設計されているので、キーワードリストに依存せず、未知のドメインにも適応します。

15 トピック × 3 ラウンドのテストで全件完走を確認済み。

## 調査の深度

テーマのリスクや鮮度要件に応じて、自動で深度が決まります。

| 深度 | 調査の深さ | 用途 |
|---|---|---|
| Quick | 主要ソース中心に簡潔に | 一般的な質問、低リスク |
| Standard | 複数ソース + 反証検索 | 技術選定、業務プロセス |
| Deep | 網羅的調査 + リスク分析 | セキュリティ、医療、法務、金融 |

## なぜこう動くのか

### 情報をどう評価しているか

Web検索の上位に表示される記事が正確とは限りません。Search Best Practice は、情報を収集する段階で3つの評価軸を適用しています。

1. **情報源の権威性** — 公式ドキュメント、学術論文、業界標準化団体の発表は、個人の体験談より信頼度が高い。情報源を「公式 → 専門機関 → 実務者 → 個人」の4段階で評価し、信頼度の高いソースを優先的に採用します
2. **情報の鮮度** — テーマによって「賞味期限」は異なります。セキュリティ対策は半年で変わりうるし、組織設計の原則は10年単位で安定している。テーマごとに適切な鮮度基準を自動判定し、基準を超えた情報には注意を付記します
3. **反証の探索** — 「これがベスト」という結論を出す前に、意図的に反対意見やアンチパターンを検索します。確証バイアス（自分の仮説に合う情報ばかり集めてしまう傾向）を構造的に排除するためです

これらは、情報リテラシー研究の SIFT（Stop, Investigate, Find, Trace）や CRAAP テスト（Currency, Relevance, Authority, Accuracy, Purpose）といった検証フレームワークを、AIの調査プロセスに組み込んだものです。

技術的なテーマだけでなく、価格設定、教育手法、組織運営など、あらゆる分野のベストプラクティス調査に同じ評価基準が適用されます。

### 「原則」と「ツール」を分離する理由

ベストプラクティスと呼ばれるものの中には、10年後も通用する知見と、来年には古くなる情報が混在しています。

たとえば「顧客データは暗号化して保管する」は原則であり、技術トレンドに左右されません。一方「AES-256 を使う」は現時点での推奨手段であり、将来より適切な方式に置き換わる可能性があります。「価格設定は顧客の支払意欲に基づく」は原則ですが、「Stripe Billing を使う」はツールの選択です。

この2つを区別せずにまとめてしまうと、ツールが更新されたときに原則ごと見直す羽目になったり、逆に時代遅れのツールに原則とセットで固執してしまったりします。分離しておけば、手段が変わっても判断の軸は残ります。

### この考え方自体が持つ価値

Search Best Practice のレポートはもちろん実務に役立ちますが、それ以上に価値があるのは、レポートが体現している **情報評価のフレームワークそのもの** です。

「発信者の権威性を確認する」「情報の鮮度を意識する」「反証を探す」「原則と手段を分けて考える」——これはAIを使わない場面でも応用できる考え方です。レポートの構造に触れることで、日常の情報収集における判断力が自然と磨かれていきます。
