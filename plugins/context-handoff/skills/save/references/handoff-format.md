# handoff.md フォーマット仕様

## 制約

- **100行以下 / 1,500トークン以下**（超過時はローテーションルールに従う）
- **git diff で復元可能な情報は書かない**（コード変更、ファイル一覧は git が持つ）
- Dead Ends の棄却理由は要約・圧縮しない

## 構造

```markdown
---
saved_at: "2026-04-06T15:30:00Z"
git_head: "abc1234"
branch: "feature/auth"
version: 1
---

## Environment
- 主要ファイル: src/auth.ts, src/session.ts
- 最後のテスト: pass (12/12)
- ビルド状態: 正常

## Dead Ends
- passport.js の session strategy → カスタムヘッダーとの互換性問題で断念
- JWT 方式 → p99 レイテンシ要件(50ms以下)を満たせず不採用

## Decisions
- セッションベース認証を採用（Redis。既にキャッシュで利用中、追加インフラ不要）
- express-session ではなく自前実装（カスタムヘッダー対応が必要なため）

## Status
- トークンリフレッシュ実装完了
- エラーハンドリング未着手

### Blocked
- デプロイ設定は DevOps チームのレビュー待ち

## Next
- src/auth.ts:45 のエラーハンドリング追加
- リトライロジック（指数バックオフ）
- 統合テストの追加
```

## セクション詳細

### YAML フロントマター

| フィールド | 必須 | 説明 |
|---|---|---|
| `saved_at` | Yes | ISO 8601 形式のタイムスタンプ |
| `git_head` | Yes | 保存時の git HEAD（短縮ハッシュ） |
| `branch` | Yes | 保存時のブランチ名 |
| `version` | Yes | フォーマットバージョン（現在は `1`） |

### セクション順序（認知科学の再没入プロセスに基づく）

1. **Environment** — 最後に触ったファイル、テスト状態、ビルド状態。再開時に「どこにいたか」を把握するための環境的手がかり（Cue Priming）
2. **Dead Ends** — 試して棄却したアプローチとその理由。**棄却理由は圧縮しない**。同じ失敗の繰り返しを防ぐ最重要セクション
3. **Decisions** — 採用した設計判断とその理由。「なぜこの形になっているか」の理解
4. **Status** — 現在の進捗。完了したこと・未着手のこと。`### Blocked` サブセクションで外部依存・待ち事項を分離
5. **Next** — 次にやるべきこと。具体的なファイルパス・行番号を含める

### L1 差分追記時のフォーマット

L1（自動追記）では、該当セクションに1行追記する:

```markdown
## Dead Ends
- passport.js → カスタムヘッダー非互換で断念
+ - express-session → 依存が重すぎるため不採用    ← L1 で追記

## Decisions
- Redis セッション採用
+ - 自前実装を選択（カスタムヘッダー対応のため）  ← L1 で追記
```

## ローテーションルール

100行を超えた場合、PostCompact（L2）整理時に以下の順で圧縮:

1. **Status**: 完了済みタスクのうち古いものを削除（直近3件は保持）
2. **Environment**: 最新状態に上書き（履歴不要）
3. **Dead Ends**: 解決済み（関連する作業が完了した）エントリを `.context/archive/` に移動
4. **Decisions**: 現在の作業ブランチに無関係なエントリを `.context/archive/` に移動（削除しない）

**削除してはいけないもの**:
- 未解決の Dead Ends（同じ失敗を繰り返す原因になる）
- 現在の作業に影響する Decisions

## archive ディレクトリの管理

`.context/archive/` には L2 整理で移動されたエントリが蓄積される。

**ファイル名**: `archive-{YYYY-MM-DD}.md`（日付ごとに1ファイル）

**肥大化防止ルール**:
- archive ファイルは **直近5件のみ保持**。6件目以降は古いものから自動削除
- L2 整理時に、archive ディレクトリ内のファイル数をチェックし、超過分を削除する
- archive は参照用であり、@import 対象ではない（コンテキスト消費なし）
