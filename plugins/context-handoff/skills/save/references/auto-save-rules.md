# CLAUDE.md に追加する自動保存ルール

## 追加する内容

プロジェクトの CLAUDE.md に以下を追記する:

```markdown
## セッション引き継ぎ

@.context/handoff.md

重要な設計判断をしたとき、アプローチを棄却したとき、.context/handoff.md の該当セクションに1行追記すること。
コンパクション後に PostCompact から更新指示が来たら全体を整理すること。
```

## 各行の役割

### `@.context/handoff.md`

CLAUDE.md の `@import` 機能で handoff.md を自動読み込みする。
- セッション開始時に自動で読み込まれる
- コンパクション後も再読み込みされる（コンパクション耐性）
- ファイルが存在しない場合は無視される

### 差分追記ルール（L1）

「重要な設計判断をしたとき、アプローチを棄却したとき」に該当セクションに1行追記する。

**トリガー条件**（明確な場合のみ。迷ったら追記しない）:
- ライブラリ・フレームワークの選定を行った
- アプローチを試して失敗・断念した
- アーキテクチャ上の方針を決めた
- ユーザーとの対話で重要な方針が確定した

**トリガーしない条件**:
- typo 修正、lint 修正、フォーマット変更
- 既存の判断の範囲内での実装作業
- テスト実行（結果が Dead End でない限り）

**追記フォーマット**: 該当セクションに1行追加
- Dead Ends: `- [試みた内容] → [棄却理由]`
- Decisions: `- [判断内容]（[理由]）`

### PostCompact 整理ルール（L2）

PostCompact フックから更新指示が来たら:
1. `.context/.pre-compact-snapshot.tmp` を読む（機械的スナップショット）
2. 既存の handoff.md の内容を基に、コンパクション後の要約と統合
3. 100行以下になるよう整理（ローテーションルールに従う）
4. Dead Ends の棄却理由は原文を保持（圧縮しない）
5. YAML フロントマターの `saved_at`、`git_head` を更新
6. 完了を1行で報告
