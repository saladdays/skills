# .claude/rules/ に配置する自動保存ルール

## 配置先

`.claude/rules/context-handoff.md` として配置する。
プロジェクトの CLAUDE.md は変更しない。

`.claude/rules/*.md` は CLAUDE.md と同じタイミングで自動読み込みされ、
コンパクション後も再読み込みされる（コンパクション耐性あり）。

## ファイルの内容

以下の内容で `.claude/rules/context-handoff.md` を作成する:

```markdown
# セッション引き継ぎ

@.context/handoff.md

重要な設計判断をしたとき、アプローチを棄却したとき、.context/handoff.md の該当セクションに1行追記すること。
追記対象: ライブラリ選定、アプローチの断念、アーキテクチャ方針の決定、ユーザーとの重要な合意。
追記しない: typo修正、lint修正、既存方針内の実装作業。迷ったら追記しない。
コンパクション後に PostCompact から更新指示が来たら、他の応答に先立ちhandoff.mdを整理すること。
```

## 各行の役割

### `@.context/handoff.md`

`@import` 機能で handoff.md を自動読み込みする。
- セッション開始時に自動で読み込まれる
- コンパクション後も再読み込みされる（コンパクション耐性）
- ファイルが存在しない場合は無視される

### 差分追記ルール（L1）

**トリガー条件**（ルールに含まれるが、詳細は以下）:
- ライブラリ・フレームワークの選定を行った
- アプローチを試して失敗・断念した
- アーキテクチャ上の方針を決めた
- ユーザーとの対話で重要な方針が確定した

**トリガーしない条件**（ルールに含まれるが、詳細は以下）:
- typo 修正、lint 修正、フォーマット変更
- 既存の判断の範囲内での実装作業
- テスト実行（結果が Dead End でない限り）

**追記フォーマット**: 該当セクションに1行追加
- Dead Ends: `- [試みた内容] → [棄却理由]`
- Decisions: `- [判断内容]（[理由]）`

### PostCompact 整理ルール（L2）

PostCompact フックから更新指示が来たら、**他の応答に先立ち**以下を実行する:
1. `.context/.pre-compact-snapshot.tmp` を読む（機械的スナップショット）
2. 既存の handoff.md の内容を基に、コンパクション後の要約と統合
3. 100行以下になるよう整理（ローテーションルールに従う）
4. Dead Ends の棄却理由は原文を保持（圧縮しない）
5. YAML フロントマターの `saved_at`、`git_head` を更新
6. 完了を1行で報告

**二重化**: PostCompact フックの `additionalContext` でも同じ指示が注入されるが、
ルールファイルにも「コンパクション後に更新指示が来たら整理すること」を含めることで、
フック注入が認識されなかった場合のフォールバックとする。

## セキュリティに関する注意

handoff.md は `@import` で毎セッション自動読み込みされるため、
悪意あるコンテンツが混入した場合にプロンプトインジェクションのベクトルになりうる。

**対策**:
- `.context/` を git 管理する場合、PR レビューで handoff.md の差分を確認する習慣を持つ
- 不審な内容（外部URLへの誘導、通常と異なる指示文）を発見した場合は handoff.md を削除して再生成する
- 共有リポジトリでは `.context/` を `.gitignore` に入れることも検討する
