# context-handoff

**AI エージェントとの作業で「前のセッションの続き」がそのまま始まる。**

セッションを跨ぐたびに同じ説明を繰り返していませんか？ 前に試してダメだったアプローチを、新しいセッションでまた試されてイライラしたことは？

context-handoff は、セッション間のコンテキストを自動で保存・復元します。保存も再開も意識する必要はありません。

## こんな方に

- AI コーディングエージェント（Claude Code、Cursor 等）を日常的に使っている
- セッションを跨ぐと「また一から説明か…」とストレスを感じている
- 前のセッションで試して失敗したことを、新セッションで繰り返された経験がある
- 複数の PC で作業し、環境を跨いで作業を引き継ぎたい

## 使い方

### 初回セットアップ

```
/context-handoff:save
```

これだけです。スキルがフック設定、CLAUDE.md へのルール追加、handoff.md の初回生成をすべて行います。

### その後は何もしなくていい

- **作業中**: AI が重要な判断をするたびに、引き継ぎメモが自動で蓄積されます
- **コンテキストが限界に達したら**: 自動的にメモが整理されます
- **新しいセッションを開始したら**: 前回の状態が自動で読み込まれ、AI が3行サマリを表示します

最初のメッセージを送る前にセッションを開始すると、AI が引き継ぎを確認してくれます。

```
AI: 前回から引き継ぎ:
    - 作業中: 認証リファクタ（src/auth.ts, src/session.ts）
    - 注意: passport.js はカスタムヘッダー非互換で断念済み
    - 次: エラーハンドリング追加
    この認識で合っていますか？

あなた: うん、続きやって
```

先にメッセージを送った場合でも、引き継ぎ情報は AI のコンテキストに読み込まれています。「前回の続きから」と伝えればいつでも引き継げます。

### 明示的に保存したいとき

```
/context-handoff:save
```

大きな区切りや、特に正確に保存したいときに使えます。普段は自動保存で十分です。

### 別の PC で続きをやるとき

```
git push   （今の PC で）
git pull   （別の PC で）
claude     （そのまま前回の続きが始まる）
```

## なぜこう動くのか

### 会話の要約ではなく「作業状態のスナップショット」

多くの AI ツールは会話を要約して引き継ぎますが、要約を重ねるとエラーが蓄積します（再帰的要約のハルシネーション問題）。context-handoff は会話を要約せず、**git に残らない情報だけ**を構造化して保存します。コードの変更内容は git が記録してくれるので、引き継ぎメモには「なぜその変更をしたか」「何を試してダメだったか」だけを書きます。

### 「やらなかったこと」の記録が最も価値がある

Chris Parnin の研究（中断されたプログラミングタスクの再開）によると、開発者が中断後に最もコストをかけるのは「同じ袋小路の再探索」です。AI エージェントには人間の Zeigarnik 効果（未完了タスクを覚えている傾向）がないため、失敗の記録がないと同じ失敗を何度でも繰り返します。context-handoff は **Dead Ends（棄却した試行）セクションを最重要として保護**し、棄却理由を圧縮しません。

### 三層の保存でどんな終わり方でも安心

| いつ | 何が起きる |
|---|---|
| 作業中に重要な判断をした | 自動で1行追記（あなたは気づかない） |
| コンテキストがいっぱいになった | 自動でメモを整理（あなたは気づかない） |
| 自分で保存したくなった | `/context-handoff:save` を叩く |

ターミナルを閉じても、PC がスリープしても、直近の状態が残っています。

## インストール手順

### Claude Code

Claude Code の画面で、以下の2行を実行するだけです。

```
/plugin marketplace add saladdays/agent-skills
/plugin install context-handoff@saladdays-skills
```

インストールできたら `/context-handoff:save` で初回セットアップが始まります。

> これは Claude Code の「プラグイン」という仕組みです。1回入れたら、あとはずっと使えます。

### Cursor

Cursor のプラグインマーケットプレイスから `context-handoff` を検索してインストール。

> [Cursor Marketplace](https://cursor.com/marketplace) に公開後はワンクリックでインストールできます。

### 他のAIツール

SKILL.md をAIツールのスキルディレクトリにコピーしてください。
